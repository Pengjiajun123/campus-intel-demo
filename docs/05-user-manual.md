# 校园情报局 Campus Intel 具体使用手册

本手册用于参赛评委、演示人员或开发者快速理解如何运行和体验 Demo。

## 1. 本地运行

进入项目目录：

```bash
cd campus-intel-demo
```

安装依赖：

```bash
npm install
```

启动开发服务器：

```bash
npm run dev
```

打开浏览器：

```text
http://localhost:3000
```

构建验证：

```bash
npm run build
```

## 2. 推荐演示路径

### 第一步：打开首页

进入首页后，先看三个信息：

1. 产品名：校园情报局 Campus Intel。
2. slogan：把 99+ 群消息，变成今天该做的三件事。
3. 今日状态贴纸：用表情和颜色表达当前校园信息压力。

建议讲解：

> 这是一个面向大学生的 AI 校园信息整理助手，用来处理群聊截图、活动海报和通知文本。

### 第二步：进入信息导入区

点击“一键体验示例”或滚动到“信息导入”区域。

可以看到三类输入：

1. 群聊截图。
2. 活动海报。
3. 通知文本。

这里有两种模式：

| 模式 | 用途 |
| --- | --- |
| 示例演示模式 | 默认模式，使用 mock 数据，适合比赛录屏和稳定演示。 |
| 真实解析模式 | 请求 `/api/analyze` 接口；未配置 `AI_API_KEY` 时自动回退 mock 数据。 |

### 第三步：使用示例素材

点击“使用示例素材”。

系统会展示四条示例输入：

1. 数据库课程群通知。
2. 奖学金班级通知。
3. 产品社团二面通知。
4. 腾讯 PCG 校园 AI 产品创意赛信息。

这些素材模拟真实大学生每天从不同渠道收到的信息。

### 第四步：开始 AI 解析

点击“开始 AI 解析”。

页面会进入 AI 解析流程，依次展示：

1. 正在识别图片和通知文本。
2. 正在提取时间、地点、材料和对象。
3. 正在判断风险和紧急程度。
4. 正在检查是否错过 deadline。
5. 正在生成今日行动建议。

说明：

> 当前 Demo 默认不强制调用真实 API，而是用稳定动画和 mock 数据展示完整流程。后续配置 `AI_API_KEY` 后，可以接入真实大模型。

### 第五步：查看情报看板

解析完成后，页面会跳到情报看板。

看板包含：

1. 已错过事项：优先补救。
2. 今日必办：必须马上处理的任务。
3. 本周机会：比赛、实习、讲座等机会。
4. 风险提醒：单独列出材料缺口、资格风险和时间风险。

卡片信息包括：

- 标题
- 类型
- 优先级
- deadline 或 eventTime
- 来源
- AI 判断理由
- 置信度
- 查看详情按钮

### 第六步：查看情报详情

点击任意卡片的“查看详情”。

详情页会展示：

1. 标题、类型、优先级、置信度。
2. deadline / eventTime / location。
3. 情报摘要。
4. AI 判断理由。
5. 来源和原文片段。
6. 下一步行动。
7. 风险提醒。

如果查看“腾讯 PCG 校园 AI 产品创意赛”，还会展示：

- 初赛提交物：Demo 链接、Demo 演示录屏、说明文档 PDF。
- 三天计划：确定选题和 PRD、完成 Demo 页面、录屏写 PDF 并测试链接。

### 第七步：生成行动计划或群回复

详情页底部有三个按钮：

1. 返回情报看板。
2. 生成行动计划。
3. 生成群回复。

这些按钮当前使用静态结果模拟 AI Agent，适合比赛录屏展示。

### 第八步：使用 AI 对话助手

回到看板下方的 AI 助手区域，可以点击快捷问题：

1. 我今晚只有 2 个小时，应该先做什么？
2. 帮我生成社团二面确认回复。
3. 帮我生成 AI 产品创意赛三天计划。
4. 我这周有哪些高风险事项？

AI 回复会结合 mock 情报卡，而不是泛泛聊天。

## 3. 状态贴纸说明

Campus Intel 用贴纸表达压力状态：

| 状态 | 含义 | 用户动作 |
| --- | --- | --- |
| 蓝色伤心 | 已错过事项 | 先补救和止损 |
| 红色愤怒 | 事情特别多且紧急 | 马上排序处理 |
| 橙色紧张 | 事情多但还有缓冲 | 分配时间块 |
| 黄色难受 | 事情少但很急 | 立即处理截止事项 |
| 粉色开心 | 事情少且可控 | 按建议推进 |
| 绿色开心 | 今日清爽 | 保持关注即可 |

状态判断逻辑：

1. 存在已过 deadline 的事项，优先显示蓝色伤心状态。
2. 高优先级事项大于等于 3，或 24 小时内截止事项大于等于 2，显示红色愤怒状态。
3. 总事项大于等于 5，但高优先级事项小于等于 2，显示橙色紧张状态。
4. 总事项小于等于 2，但存在 24 小时内截止事项，显示黄色难受状态。
5. 总事项 1 到 3，且没有高风险事项，显示粉色开心状态。
6. 没有今日必办事项，只有低优先级机会或提醒，显示绿色开心状态。

## 4. 真实 AI 接口说明

接口路径：

```text
POST /api/analyze
```

请求体示例：

```json
{
  "mode": "real",
  "text": "今晚 22:00 前回复社团二面确认",
  "files": [
    {
      "name": "poster.png",
      "type": "image/png",
      "size": 12345,
      "source": "活动海报"
    }
  ]
}
```

默认行为：

- `mode` 为 `mock` 时，返回本地 mock 数据。
- `mode` 为 `real` 但没有 `AI_API_KEY` 时，自动回退 mock 数据。
- `mode` 为 `real` 且配置 `AI_API_KEY` 时，可调用真实大模型。

可选环境变量：

```text
AI_API_KEY=你的真实 API Key
AI_API_BASE_URL=https://api.openai.com/v1/chat/completions
AI_MODEL=gpt-4o-mini
```

安全说明：

- 项目代码中不写死任何 API Key。
- 比赛 Demo 阶段不强制依赖真实 API。
- 即使真实接口失败，也会回退到 mock 数据，保证演示稳定。

## 5. 常见问题

### 为什么默认不直接调用真实 AI？

比赛 Demo 最重要的是稳定演示。真实 API 可能受到网络、额度、模型响应时间影响，因此默认使用示例演示模式。

### 这是不是普通待办工具？

不是。普通待办工具要求用户自己输入任务。Campus Intel 的核心是把非结构化校园信息自动解析成结构化情报，再判断风险和行动顺序。

### 为什么要保留来源和原文片段？

AI 生成的建议需要可信依据。`source` 和 `sourceQuote` 可以让用户知道提醒来自哪里，避免 AI 输出变成不可追溯的黑箱。

### 演示时最推荐点哪张详情卡？

推荐点“腾讯 PCG 校园 AI 产品创意赛”，因为它和本次参赛场景绑定最强，也能展示提交物拆解和三天计划。

## 6. 验收清单

演示前请确认：

- 首页能打开。
- “一键体验示例”能跳到导入区。
- “使用示例素材”能显示四条素材。
- “开始 AI 解析”能进入解析动画。
- 解析完成后能展示情报看板。
- 卡片能跳转到详情页。
- 详情页能生成行动计划和群回复。
- AI 助手快捷问题能显示对应回复。
- `npm run build` 能通过。

## 7. 适合放进 PDF 的使用总结

评委只需要按一个路径体验：打开首页，点击一键体验示例，加载示例素材，开始 AI 解析，查看情报看板，进入比赛详情页，最后询问 AI 助手“我今晚只有 2 个小时应该先做什么”。这个路径能完整展示 Campus Intel 从混乱校园信息到清晰行动建议的核心价值。
