# 校园情报局 Campus Intel 初赛最终提交检查清单

本清单按照初赛提交指南整理，用于录屏前、导出 PDF 前和正式提交前逐项核对。

## 1. 三项必交材料

| 提交物 | 格式要求 | 当前项目状态 | 录屏前处理 |
| --- | --- | --- | --- |
| Demo 链接 | 可访问的 URL 链接，评委能直接打开体验 | 本地 Demo 可运行；还需要部署成公网 URL | 录屏可先用本地 `http://localhost:3000`，正式提交前必须替换为公网链接 |
| Demo 演示录屏 | MP4，H.264 编码，时长不超过 3 分钟，大小不超过 500MB，分辨率不低于 1080p | 已准备 3 分钟讲稿和演示路径 | 按 `docs/03-recording-script.md` 录制 |
| 说明文档 | PDF，大小不超过 50MB | 已准备 PDF 正文草稿 | 使用 `docs/07-submission-pdf-draft.md` 整理为 PDF |

## 2. Demo 链接提交规范

正式提交时，Demo 链接必须满足：

- 可以在浏览器中直接打开。
- 不依赖本地服务、内网地址或登录态。
- 如果需要登录，必须在说明文档中提供体验账号；当前 Demo 不需要登录。
- 提交截止日前后和评审期间都应保持可访问。
- 链接失效会影响评分。

### 当前建议

录屏阶段可以使用：

```text
http://localhost:3000
```

正式提交阶段建议部署到 Vercel、Netlify 或其他静态/Node 托管平台，并将公网地址填入 PDF：

```text
Demo 链接：待部署后填写
```

## 3. 演示录屏规范

| 项目 | 要求 |
| --- | --- |
| 视频时长 | 不超过 3 分钟 |
| 视频格式 | MP4，H.264 编码 |
| 分辨率 | 不低于 1080p |
| 文件大小 | 不超过 500MB |
| 语音讲解 | 必须包含语音讲解或字幕说明，清晰演示产品思路和核心功能 |
| 命名建议 | `选手名_赛道_作品名_Demo演示.mp4` |

## 4. 推荐录屏结构

录屏总时长建议 2 分 40 秒到 2 分 55 秒，给最后剪辑留余量。

| 时间 | 内容 | 画面 | 重点 |
| --- | --- | --- | --- |
| 0:00 - 0:30 | 开场介绍 | 首页 Hero | 一句话说明产品是什么、解决谁的问题 |
| 0:30 - 1:30 | 核心流程演示 | 导入区 + AI 解析流程 | 展示输入、示例素材、解析步骤 |
| 1:30 - 2:30 | 亮点与总结 | 情报看板 + 详情页 + AI 助手 | 展示 AI 判断、来源追溯、行动建议 |
| 2:30 - 3:00 | 落地价值 | 看板或 AI 助手 | 总结差异化、落地和商业化 |

详细讲稿见：

```text
docs/03-recording-script.md
```

## 5. 录屏前操作路径

建议按照这条路径录：

1. 打开首页。
2. 讲 slogan：把 99+ 群消息，变成今天该做的三件事。
3. 点击“一键体验示例”或滚动到信息导入区。
4. 点击“使用示例素材”。
5. 简单展示四条示例素材。
6. 点击“开始 AI 解析”。
7. 等待解析动画完成。
8. 展示情报看板。
9. 重点讲“已错过事项”“今日必办”“本周机会”“风险提醒”。
10. 点击“腾讯 PCG 校园 AI 产品创意赛”详情。
11. 展示提交物和三天计划。
12. 点击“生成行动计划”。
13. 返回看板，滚动到 AI 助手。
14. 点击“我今晚只有 2 个小时，应该先做什么？”
15. 用一句话总结落地价值。

## 6. 录屏前技术检查

录屏前运行：

```bash
npm run build
npm run dev
```

打开：

```text
http://localhost:3000
```

确认：

- 首页能打开。
- “校园 / 情报局”标题没有断字错误。
- 今日状态贴纸居中。
- 点击贴纸有弹跳反馈。
- “一键体验示例”能跳到 Demo 区域。
- “使用示例素材”能显示四条素材。
- “开始 AI 解析”能进入解析流程。
- 解析完成后看板出现。
- 卡片详情页能打开。
- “生成行动计划”和“生成群回复”有静态结果。
- AI 助手快捷问题能显示对应回复。

## 7. 说明文档 PDF 结构

PDF 建议控制在 8 到 12 页，不要堆太多截图。结构如下：

| 模块 | 内容 |
| --- | --- |
| 封面页 | 作品名称、参赛赛道、选手姓名、Demo 链接 |
| 模块一：用户洞察与问题定义 | 目标用户、用户痛点、使用场景 |
| 模块二：产品方案设计 | 产品概述、核心功能、功能架构图、交互流程、创新差异化 |
| 模块三：AI 原生能力说明 | AI 核心能力、AI 如何解决痛点、AI 技术方案 |
| 模块四：加分项 | 落地可行性、商业化思考 |
| 附录 | 评分对照、演示路径、构建验证 |

正文草稿见：

```text
docs/07-submission-pdf-draft.md
```
