## 模块一：用户洞察与问题定义

### 1. 目标用户

核心用户是普通大学生，尤其是同时处理课程、班级事务、社团、比赛、实习和讲座信息的学生。

他们每天面对的信息来源包括：

- 课程群：作业、实验报告、考试安排、课堂材料。
- 班级群：奖学金、评优、材料收集、学院通知。
- 社团群：招新、面试、活动确认。
- 比赛通知：报名、提交物、录屏、说明文档。
- 实习群：内推、简历投递、岗位信息。
- 公众号和海报：讲座、活动、校园资源。

### 2. 核心痛点

大学生不是缺少信息，而是被信息淹没。

典型痛点包括：

1. 信息分散在多个群和平台中，难以集中查看。
2. 群聊消息数量大，重要通知容易被闲聊淹没。
3. 不同通知的后果不同，有的影响成绩，有的影响资格，有的只是机会提醒。
4. 用户看完通知后，仍然不知道今晚应该先做什么。
5. deadline 一旦错过，用户往往缺少补救提醒和止损建议。

### 3. 典型场景

某天晚上，学生小林打开手机，发现自己同时收到：

- 数据库课程群要求本周五前提交实验报告。
- 班级群要求周三中午前填写奖学金信息统计表。
- 产品社团群要求今晚 22:00 前回复是否参加二面。
- 公众号推送腾讯 PCG 校园 AI 产品创意赛。
- 实习交流群发布产品经理实习内推。
- 图书馆系统提示借书已经逾期。

如果小林只靠自己翻群消息，很容易先看有趣的讲座或比赛，却忽略真正紧急的二面确认和已逾期事项。

### 4. 问题定义

本项目要解决的问题是：

> 如何让 AI 从混乱的校园信息中识别真正重要的事项，并把它们整理成学生今天能直接执行的行动顺序？

## 模块二：产品方案设计

### 1. 产品概述

校园情报局 Campus Intel 是一个 AI 校园信息整理助手。

它的输入是碎片化信息：

- 群聊截图
- 活动海报
- 通知文本
- 比赛要求
- 实习内推

它的输出是结构化情报：

- 情报卡片
- 今日必办
- 风险提醒
- 已错过事项
- 本周机会
- AI 行动建议

### 2. 核心功能

#### 功能一：信息导入

用户可以模拟上传群聊截图、活动海报，或粘贴通知文本。为了保证比赛 Demo 稳定，系统提供“使用示例素材”按钮，一键加载内置校园信息。

#### 功能二：AI 解析流程

AI 解析流程包括 5 步：

1. 识别群聊截图和通知文本。
2. 提取 deadline、地点、材料和对象。
3. 判断事项类型和紧急程度。
4. 匹配课程、兴趣和校园身份。
5. 生成今日行动清单。

#### 功能三：情报看板

情报看板分为：

- 已错过事项
- 今日必办
- 本周机会
- 风险提醒

每张情报卡片展示标题、类型、优先级、deadline、来源、AI 判断理由、置信度和详情入口。

#### 功能四：情报详情页

详情页展示：

- 标题
- 类型、优先级、置信度
- deadline / eventTime / location
- 情报摘要
- AI 判断理由
- 来源和原文片段
- 下一步行动
- 风险提醒

如果是腾讯 PCG 校园 AI 产品创意赛，还会额外展示提交物和三天计划。

#### 功能五：AI 对话助手

AI 助手提供快捷问题：

- 我今晚只有 2 个小时，应该先做什么？
- 帮我生成社团二面确认回复。
- 帮我生成 AI 产品创意赛三天计划。
- 我这周有哪些高风险事项？

AI 回复会结合情报卡片，而不是泛泛聊天。

### 3. 产品信息架构

```text
首页
  → 信息导入
    → 使用示例素材 / 上传信息
    → AI 解析流程
      → 情报看板
        → 已错过事项
        → 今日必办
        → 本周机会
        → 风险提醒
      → 情报详情页
        → 来源追溯
        → AI 判断理由
        → 下一步行动
      → AI 对话助手
        → 优先级建议
        → 回复生成
        → 比赛计划
```

### 4. 交互流程

用户只需要完成一个简单路径：

```text
打开 Demo
  → 点击一键体验示例
  → 使用示例素材
  → 开始 AI 解析
  → 查看情报看板
  → 进入详情页
  → 生成行动计划或群回复
  → 向 AI 助手提问
```

### 5. 创新与差异化

校园情报局的创新点不是把待办列表做得更好看，而是引入“校园压力状态”的表达方式。

状态贴纸含义：

| 状态 | 含义 |
| --- | --- |
| 蓝色伤心 | 已错过事项，需要先补救 |
| 红色愤怒 | 事情特别多且紧急 |
| 橙色紧张 | 任务堆积但还有缓冲 |
| 黄色难受 | 数量不多但截止逼近 |
| 粉色开心 | 当前状态轻松可控 |
| 绿色开心 | 今日清爽、安全 |

这里颜色优先表达“压力状态”，而不是固定表达信息类型。这样用户一眼就能知道当前最该处理什么。

## 模块三：AI 原生能力说明

### 1. AI 核心能力

Campus Intel 的 AI 能力包括：

- 多源信息理解：处理群聊截图、海报和通知文本。
- 关键字段抽取：识别 deadline、eventTime、location、source、materials。
- 风险判断：判断是否影响成绩、资格、面试、提交物或机会窗口。
- 优先级排序：区分已错过、高优先级、中优先级和低优先级。
- 来源追溯：保留 source 和 sourceQuote，让用户知道 AI 判断依据。
- 行动生成：生成 nextActions、risks、群回复和比赛计划。

### 2. AI 如何解决用户痛点

| 用户痛点 | AI 解决方式 |
| --- | --- |
| 群消息太多 | 从长文本或截图 OCR 中抽取真正有行动意义的事项 |
| 不知道先做什么 | 根据 deadline、风险和优先级生成今日必办 |
| 害怕漏掉重要信息 | 单独展示风险提醒和已错过事项 |
| 不信任 AI 总结 | 保留来源和原文片段 |
| 行动成本高 | 直接生成下一步行动、群回复和三天计划 |

### 3. AI 工作流

```text
非结构化校园信息
  → OCR / 文本输入
  → LLM 结构化抽取
  → JSON 情报卡片
  → 风险与优先级规则
  → 看板展示
  → AI 助手生成行动建议
```

### 4. 结构化输出设计

每张情报卡片包含：

- `id`
- `title`
- `category`
- `priority`
- `deadline`
- `eventTime`
- `location`
- `source`
- `sourceQuote`
- `confidence`
- `summary`
- `aiReason`
- `nextActions`
- `risks`
- `tags`
- `matchScore`

这说明项目不是简单调用聊天 API，而是要求 AI 输出可被产品界面直接消费的结构化 JSON。

### 5. API 预留

项目已预留：

```text
POST /api/analyze
```

默认行为：

- 没有配置 `AI_API_KEY` 时返回 mock 数据，保证 Demo 稳定。
- 配置 `AI_API_KEY` 后，可调用真实大模型。
- 代码中不写死任何 API Key。

相关文件：

- `src/app/api/analyze/route.ts`
- `src/lib/analyzePrompt.ts`
- `src/lib/types.ts`
- `src/data/mockIntel.ts`

## 模块四：落地可行性与商业化思考

### 1. 技术可行性

当前 Demo 使用：

- Next.js
- TypeScript
- Tailwind CSS
- 本地 mock 数据
- 预留 AI API

技术路线轻量，适合快速迭代和参赛展示。

### 2. 后续落地路线

| 阶段 | 能力 | 说明 |
| --- | --- | --- |
| Demo 阶段 | 前端体验 + mock 数据 | 保证评委打开即可体验 |
| MVP 阶段 | OCR + LLM | 支持截图识别和真实文本解析 |
| 校园试用阶段 | 日历 / 提醒 / 复制回复 | 帮助学生真正完成行动 |
| 产品化阶段 | 学生端 + 组织端 | 面向个人效率和校园通知管理 |

### 3. 商业化方向

#### ToC 学生个人版

基础解析免费，高级功能包括长期日历同步、风险复盘、个性化机会推荐。

#### 校园组织版

面向社团、学生会、学院，帮助他们发布结构化通知，并提升学生对 deadline 和材料要求的理解率。

#### 赛事和活动分发

比赛、讲座、实习项目可以通过结构化卡片触达目标学生，提高报名和参与转化。

#### 职业发展服务

结合学生兴趣画像，把比赛、实习、讲座和项目经历串成成长路径。

### 4. 长期价值

校园情报局可以成为大学生的校园信息入口。它不是替代群聊，而是在群聊和用户行动之间加一层 AI 理解层，让学生从“被消息推着走”变成“按行动优先级处理校园生活”。
